Amigos, vocês estão a complicar o que não tem complicação. Para passar a cumprir o RNID, no caso da declaração do IRS (que é naquilo que o meu comentário incidiu), não é preciso "começar do zero", reformular a estrutura existente nem nada disso. É simplesmente necessário criar uma nova interface (não é sequer preciso descontinuar as existentes, incluindo a interface Java!), usando normas abertas. Como o +Pedro Calção referiu, isso até já foi feito por um terceiro (Modelo3.pt), não há razão para as finanças não fazerem o mesmo. BTW, enquanto o WCAG 1.0 dizia que era obrigatória a existência de "alternatives" para coisas como Javascript, o WCAG 2.0 já não o faz. Nada impede o modelo3.pt de ser HTML 4.01 ou XHTML + CSS 2.1 + Javascript e cumprir o WCAG 2.0 AA (as normas obrigatórias pelo RNID). Não sei se o modelo3.pt em particular cumpre essas normas ou não (não tenho lá conta), mas mesmo que não cumpra não há razões para que as finanças não façam algo similar, e que cumpra.
